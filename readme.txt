

                          Travail pratique 4
          ---créer un plan de sécurisation des mots de passe---

Description

Mon plan de sécurisation des mots de passe présentant 2 approches

 1- Approche: exiger des mots de passe plus longs (comportant un minimum de 8 caractères) 
et ne pas imposer de longueur maximale 

2-Établir une liste noire de mots de passe: ma list noir contient 2 mot de passe bannis 123456789 et motdepasse

J'ai utiliser php dans ce projet ainsi pour hacher le mot de passe

ensuite j'ai utilisé Regex pour que le mot de passe contient au moins huit caractères, au moins une lettre majuscule, une lettre minuscule et un chiffre EX: Abcdef123

A la fin j'ai sauvegardé les mots de passe dans une base de données et retournez un message de connexion si la combinaison utilisateur/mot de passe existe déjà dans la base de données.
